## 1. 한 줄 요약

> **"요즘 뭐가 핫해?"** 40대 이후 사용자가 세대/사회 흐름을 3분 안에 따라잡을 수 있는
> **트렌드 디스커버리 서비스** — 단순 순위가 아니라 "왜 뜨는지"를 같이 제공.

---

## 2. 핵심 인사이트 (타겟 페르소나 = 40+)

기존 트렌드 서비스(네이버 데이터랩·썸트렌드·구글 트렌드)가 **20~30대의 분석 도구**로 만들어져 있어
40대 이후에게는 세 가지 장벽이 있다:

| 장벽              | 증상                                     | 설계 대응                          |
| ----------------- | ---------------------------------------- | ---------------------------------- |
| **해석 피로**     | 랭킹만 보여주고 "그게 뭔지" 설명 없음    | 각 키워드에 3줄 맥락 요약 (LLM)    |
| **관심사 미스핏** | 알고리즘이 Z세대 편향                    | 40+ 가중치 피드 + 본인 관심사 토글 |
| **공유 마찰**     | 자녀/친구와 대화 소재로 쓰고 싶은데 복잡 | 카톡 원클릭 이미지 카드            |

**Jobs-to-be-Done**:

- J1. 자녀/배우자와 대화 소재를 얻고 싶다 (관계 유지)
- J2. 직장/모임에서 "그거 몰라?" 소외감 피하고 싶다 (사회적 자존감)
- J3. 세상이 어디로 흐르는지 감을 잡고 싶다 (불안 해소)

이 3개 JTBD는 모두 **"5분 안에 가볍게"** 충족돼야 한다 — 심층 리포트 아님.

## 3. 경쟁 포지셔닝

| 서비스             | 강점        | 40+ 타겟에서 약점               |
| ------------------ | ----------- | ------------------------------- |
| 네이버 데이터랩    | 공식 데이터 | UI가 개발자/마케터용, 해석 없음 |
| 썸트렌드           | 감성 분석   | B2B 가격, 심층 지표             |
| 구글 트렌드        | 글로벌      | 한국 맥락 부족, 설명 없음       |
| 카드뉴스(픽기)     | 포맷 친숙   | 트렌드 특화 아님, 업데이트 느림 |
| 너목보·유튜브 쇼츠 | 소비 쉬움   | 취사선택 강제 불가, 피로도      |

**Hot Keyword의 포지션**: "**카드뉴스의 가독성 × 네이버 데이터랩의 신뢰성 × LLM 맥락 요약**"
단일 제품이 이 교집합을 노린 사례는 아직 없다고 판단.

## 4. 서비스 차별화 3축

1. **40+ 친화 UI** — 큰 글자(기본 17px+), 단일 컬럼, 이미지 우선, 카카오톡 공유 1-tap
2. **"왜 뜨는지" 맥락 요약** — 각 키워드마다 LLM이 생성한 3줄 스토리 + 관련 뉴스 3개
3. **세대 비교 뷰** — "오늘 20대가 검색한 것 vs 50대가 검색한 것" (대화 소재 + 세대 이해)

## 5. MVP 기능 (우선순위 순)

| #   | 기능                   | 설명                                                   | 난이도 |
| --- | ---------------------- | ------------------------------------------------------ | ------ |
| F1  | **오늘의 핫 TOP 10**   | 카테고리 5종 (연예·사회·건강·경제·IT) × 일일 갱신      | L      |
| F2  | **3줄 맥락 요약**      | 각 키워드 클릭 → LLM이 생성한 "누가/왜/어디서"         | M      |
| F3  | **세대 비교**          | 20대 TOP / 50대 TOP 나란히 (스와이프 전환)             | M      |
| F4  | **관심사 토글**        | 초기 3-4개 선택 → 개인화 피드 (계정 없이 localStorage) | L      |
| F5  | **카카오톡 공유 카드** | 선택한 키워드를 1장짜리 이미지로 생성 → 1-tap 공유     | M      |
| F6  | **주간 다이제스트**    | 이메일/알림톡으로 주 1회 "이번 주 핫 7" 발송           | M      |

**MVP 배제**:

- 댓글/커뮤니티 (운영 부담, 타겟에게 매력 약함)
- 실시간 알림 (피로, 40+는 아침 1회 소비 패턴)
- 깊이 있는 보고서 (제품 핵심과 모순)

## 6. 정보 구조 (IA) — 모바일 웹 기준

```
[랜딩]
  └─ "오늘 핫한 TOP 10" (카테고리 탭 5개)
       ├─ 각 키워드 카드 → 클릭
       │   └─ 상세: 3줄 요약 + 관련 뉴스 3개 + 공유 버튼
       ├─ [세대 비교] 스와이프
       ├─ [내 관심사 설정]
       └─ [주간 다이제스트 구독] (이메일/카카오)
```

한 손가락으로 3회 스크롤 안에 모든 걸 볼 수 있는 깊이.

## 8. 기술 스택 — GoldenCheck 인프라 90% 재활용

| 레이어 | 재활용                                                                   | 신규                                                               |
| ------ | ------------------------------------------------------------------------ | ------------------------------------------------------------------ |
| DB     | Postgres (신규 스키마 `hotkeyword_*`)                                    | 랭킹 스냅샷 테이블                                                 |
| 큐     | BullMQ `queue:batch` 재활용                                              | `queue:hotkeyword` 분리 고려                                       |
| 크롤러 | Category A/B 패턴                                                        | 구글트렌드·YouTube 신규                                            |
| LLM    | 기존 LLM 클라이언트 래퍼                                                 | 경량 요약 프롬프트 (#01 Macro View 변형)                           |
| 프론트 | React + Vite (별도 앱 or 라우트)                                         | 모바일 우선 컴포넌트 세트                                          |
| 인증   | **사용 안 함** (GoldenCheck JWT 미들웨어 `/hotkeyword/*` 경로 제외 처리) | 다이제스트 구독 시에만 이메일 캡처 (newsletter_subscribers 테이블) |
| 인프라 | Railway (기존 서비스 or 신규)                                            | Cloudflare Image (공유 카드 렌더)                                  |

**데이터 카테고리 분리 원칙**: 기존 A/B 원칙 유지 — HotKeyword는 **신규 카테고리 C (Public Trend)** 로 선언, 쓰기 롤 분리.

## 9. 프론트엔드 UX 원칙 (40+ 특화)

1. **기본 폰트 17px**, 본문 16px 이하 금지 (WCAG AAA 근접)
2. **단일 컬럼**, 좌우 스와이프 없이 세로 스크롤로 완결
3. **1화면 1주제** — 3줄 요약 카드는 화면 채움
4. **CTA는 녹색 큰 버튼** — "공유하기" 등 핵심 동사 중심
5. **설명 없는 아이콘 금지** — 아이콘 옆 한글 라벨 병기
6. **다크모드 자동 대응** (GoldenCheck 패턴 재사용)
7. **광고 배치**: 관심사 피드 중간 1곳 (네이티브) — 과다 피하기

## 10. 콘텐츠 포맷 명세

### 10.1 키워드 카드 (목록 뷰)

```
┌─────────────────────────────────┐
│ 1위  🔥 "당근마켓 홍콩"           │
│                                 │
│ 홍콩 당근 서비스 런칭 이슈       │
│ → 해외 진출 논쟁 쟁점화           │
│                                 │
│ 💬 오늘 12,300건 언급            │
│ [자세히] [공유]                   │
└─────────────────────────────────┘
```

### 10.2 상세 페이지

- Who/What/Why/So-What 4 섹션
- 관련 뉴스 3건 (썸네일 + 제목만, 외부 링크)
- 하단 "다른 연령대가 본 것" 추천 3건

### 10.3 카카오 공유 카드 (1200×630 이미지)

- 상단 로고 바
- 중앙 키워드(대형) + 3줄 요약
- 하단 출처 + "더보기 → goldencheck.kr/hotkeyword"
- 서버 사이드 Puppeteer 렌더링

## 13. MVP 로드맵 (6주, Phase α~γ)

### Phase α — 데이터 (2주)

- 크롤러 3종 확장 (네이버 실시간·구글 트렌드·YouTube)
- `hotkeyword_*` 스키마 + BullMQ 배치 등록
- 일 1회 랭킹 스냅샷 확정

### Phase β — 콘텐츠 + 프론트 (2주)

- LLM 요약 프롬프트 (3줄 요약, 환각 방지 가드)
- 모바일 웹 MVP: 랜딩 + 상세 + 관심사 토글
- 카카오 공유 카드 렌더링

### Phase γ — 리텐션 (2주)

- 주간 다이제스트 이메일 (`resend` 재활용)
- 세대 비교 뷰
- 간단 이벤트 트래킹 (재방문율, 공유율)

**총 예상 공수**: 1 풀스택 + 0.3 디자인 × 6주

## 15. 결정 이후 개방 질문 (다음 논의 대상)

D1~D5가 확정됐으므로 다음 라운드에서 논의할 항목은 실행 세부사항이다.

1. **광고 인벤토리 파트너** — 카카오 AdFit / Google AdSense / 네이버 자체 중 어느 것부터?
2. **일일 배치 시각** — 07:00 KST 확정 전에 데이터 소스의 최신화 시점 재확인 필요 (네이버 06:00 / 구글 실시간 / 유튜브 06~07시 KST)
3. **카테고리 5개 구체화** — "연예·사회·건강·경제·IT"의 세부 범위 정의 (예: "정치"는 사회에 포함? 별도?)
4. **공유 카드 포맷 옵션** — 정사각 (인스타) vs 1200×630 (카톡·페북) vs 둘 다 생성
5. **GDPR·개인정보** — 무인증이지만 쿠키 기반 client_id + 광고 식별자 → 개인정보처리방침 보강 필요
6. **Category C 롤 정의** — 기존 `batch_worker` / `signalcraft_worker`와 분리할 `trend_worker` 롤의 권한 범위
